# **Senior Software Engineer (Frontend)**

Thank you for your interest in joining iBox Lab. Below is a practical take-home exercise. This exercise is intentionally open-ended. We are interested in how you approach system design, scalability, reliability, maintainability, and engineering trade-offs, not just whether the code works.

**Time expectation:** 4 to 8 hours.
Please do **not** over-build. We value architectural judgment and correct fundamentals over breadth. If you run out of time, prioritize, and clearly document what you'd do next and why.

**Submission deadline:** 72 Hours after receiving this assignment.

## The Scenario - Flight Search Aggregator

You are building the frontend experience for a flight search and booking application. A user searches for flights, reviews available options, filters and sorts results, selects a flight, and completes a booking flow. The application should be designed with real-world considerations in mind, including usability, performance, scalability, and maintainability. You are free to make implementation decisions that you believe best support those goals.

## Provided mock data / API

You do **not** need to integrate with a real backend. Use local mock data, a mock API, or any approach you prefer. Create a `flights.json` or mock API(Recommended) with ~30+ flights across several airlines for one route, airline, flight number, departure/arrival, duration, stops, price.

## What to build - **Requirements**

1. **Flight Search**
   - Search for flights using origin, destination, date, and passenger count.
   - View search results in a clear and efficient format.
   - Sort results using different criteria.
   - Filter results using relevant attributes.
   - Understand the current state of the application during loading, empty, and error conditions.

   The exact layout and user experience are up to you.

2. **Flight Selection & Booking**
   - Select a flight from the search results.
   - Review flight information before booking.
   - Complete a booking form.
   - Receive a booking confirmation after successful submission.

   Include appropriate validation and user feedback throughout the booking flow.

## Engineering Expectations

- Use React, Next.js, and TypeScript.
- Structure the application in a maintainable way.
- Keep responsibilities clearly separated.
- Organize components and application logic thoughtfully.
- Design a state management approach that you believe is appropriate for this application.
- Reusability
- Tests Coverage
- Documentation: Provide `README.md` and `ARCHITECTURE.md`
